# ספריית פרומפטים

תבניות פרומפט מוכנות לפרודקשן לתהליכי עבודה נפוצים של SDLC.

## מדריך מהיר

| שלב SDLC | פרומפט | מתי להשתמש |
| ------------------ | ---------------------------------------------------------------------- | ------------------------------------------------ |
| **Onboarding** | [צור AGENTS.md](/prompts/onboarding/generate-agents-md) | יצירת קבצי קונטקסט לפרויקט באופן אוטומטי |
| **Specifications** | [צור System Spec](/prompts/specifications/generate-spec) | לפני שינוי מערכות מורכבות או תכנון פיצ'רים |
| **Specifications** | [תבנית Spec](/prompts/specifications/spec-template) | כתיבת specs מאפס לפיצ'רים חדשים או design docs |
| **Planning** | [גילוי Edge Cases](/prompts/testing/edge-case-discovery) | לפני יישום פיצ'רים או כתיבת בדיקות |
| **Implementation** | [Debug מבוסס-ראיות](/prompts/debugging/evidence-based-debug) | כשמדבגים בעיות או התנהגות לא צפויה |
| **Testing** | [אבחון כישלון בדיקות](/prompts/testing/test-failure-diagnosis) | כשבדיקות נכשלות במהלך פיתוח או CI/CD |
| **Review** | [סקירת קוד מקיפה](/prompts/code-review/comprehensive-review) | לפני commit לשינויים או הגשת PRs |
| **Pull Requests** | [מחולל תיאור PR](/prompts/pull-requests/dual-optimized-pr) | כשיוצרים pull requests לסקירה |
| **Pull Requests** | [סקירת PR בסיוע AI](/prompts/pull-requests/ai-assisted-review) | כשסוקרים pull requests שהוגשו על ידי אחרים |

## שימוש

התאימו placeholders ודוגמאות domain-specific לקונטקסט שלכם. וודאו את הפלט של הסוכן באמצעות ראיות.

## תרומה

הפרומפטים האלה מתפתחים על בסיס שימוש אמיתי. נשמח לקבל תרומות לשיפורים או התאמות domain-specific.
