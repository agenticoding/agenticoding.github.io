AI agents make execution cheap. They also make production judgment more valuable.

An agent can search, edit, run commands, and produce convincing artifacts faster than you can trust it. The bottleneck is no longer typing. It is deciding what the agent needs to know, what it may change, and what makes it trustworthy enough to ship.

**Agentic Coding is a technical reference for operating coding agents on production work.** It turns those decisions into a repeatable system: ground the agent in reality, bound the work, isolate execution from judgment, and require evidence before acceptance.

The goal is not autonomous code generation. It is controlled leverage: faster execution without transferring architecture, product, security, or release ownership to a model.

## The operating method

### Give agents reality

Useful context is selected, not dumped. You will learn to find the code, constraints, documentation, and decisions that define the task; choose the **largest reliable unit of work**; and manage context with a recurring cycle: **Write, Select, Isolate, Compress**.

### Give agents boundaries

A request becomes operable when scope, contracts, stop conditions, and checkpoints are explicit. The book uses a four-phase loop—ground, plan, execute, validate—and shows where phases can run concurrently while keeping one context's framing from carrying directly into another.

### Demand independent evidence

Implementation is a candidate, not a conclusion. You will derive tests in fresh contexts, route failures back to the phase that caused them, and move truth from temporary plans into durable specs, tests, code, and release evidence. Over time, the specification and the system converge.

## Start with your current problem

You can read the book sequentially or enter where the work is stuck:

- **The model is unpredictable:** start with [LLMs Demystified](/how-llms-work), [Agents Demystified](/how-agents-work), and [Prompting 101](/prompting-101).
- **The agent lacks codebase reality:** go to [Context Engineering](/context-engineering).
- **A task is too large or unreliable:** use the [Four-Phase Workflow](/high-level-methodology) and [Reliability Levers](/reliability-levers).
- **A feature request is still ambiguous:** turn it into an executable contract with [Spec-Driven Development](/spec-driven-development).
- **A diff looks plausible but is not yet trustworthy:** derive evidence with [Validation](/validation).
- **Agent output degrades as the codebase grows:** read [Writing Agent-Friendly Code](/agent-friendly-code).

## Who this is for

This book is for software engineers, product designers, project managers, and product managers who use coding agents on production work. It treats each as an operator: the relevant context, decisions, and evidence change by role, but responsibility for the outcome does not.

This is an advanced guide, not a beginner programming course, prompt library, or recipe for hands-off software generation. Production-level agent work requires basic software engineering fluency. You are expected to recognize most of the engineering concepts used here, follow the technical implications, and fill any gaps on your own—not to be a software engineer by title.

You do not need ML expertise, a preferred model, or a specific harness. You do need enough judgment in your discipline to recognize when the work is incomplete, unsafe, or wrong.

## Where the method comes from

The book is written by [Ofri Wolfus](/about), an ex-Google engineer and Senior Software Architect with more than 20 years across embedded systems, cloud infrastructure, developer tools, and LLM applications. Its methods grew from production systems, lessons from hundreds of engineers using modern AI tools, and a deliberately inspectable experiment: building [ChunkHound](https://chunkhound.github.io)—including its implementation—through coding agents.

ChunkHound is open source under the MIT license and handles codebases above 100 million lines. It appears throughout the book as one reference implementation of grounding, research pipelines, and context isolation—not as an exclusive recommendation. The [book source](https://github.com/agenticoding/agenticoding.github.io) and [ChunkHound source](https://github.com/chunkhound/chunkhound) are available for inspection.
